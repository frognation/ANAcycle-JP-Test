# Copilot Instructions (ANAcycle-JP-Metaball)

이 폴더(ANAcycle-JP-Metaball) 내부 작업에 대해, **중요한 예외를 제외하고는 사용자 컨펌 없이 바로 진행**한다.

## 기본 원칙
- 목표: 작업 흐름 끊김 없이 빠르게 구현/수정.
- 판단이 애매하면: **가장 단순한 해석으로 먼저 구현**하고, 필요할 때만 1~2개의 짧은 질문.
- 범위: 이 폴더 내부(하위 폴더 포함)에서의 파일 추가/수정/복제/정리.

## 컨펌 없이 진행해도 되는 것(기본 허용)
- UI/레이아웃 수정(패널 추가, 섹션 재정렬, 텍스트/라벨 변경, 값 표시 방식 변경 등)
- 새 페이지/새 효과 탭 추가(기존 페이지 복제 기반)
- CSS/JS 리팩터(동작 동일 유지 + 명확한 개선)
- 로컬 개발 서버 실행(포트 충돌 자동 회피)
- 소규모 폴더/파일 이동 및 이름 변경(이 폴더 내부에서만)
- 이미지/폰트/정적 자산 경로 수정 및 링크 복구

## 컨펌이 필요한 것(중요 예외)
아래는 반드시 실행 전에 확인 질문 1번 하고 진행:
- 대량 삭제/정리: 10개 이상 파일 삭제, 또는 폴더 통째 삭제
- 되돌리기 어려운 변경: 빌드/배포 설정 변경, 루트 `index.html` 동작을 바꾸는 리다이렉트 제거/변경
- 외부로 영향: GitHub 푸시/PR 생성/커밋, 외부 서비스 연결, 네트워크로 업로드
- 데이터 손실 위험: 기존 결과물을 덮어쓰는 작업(백업 없이 overwrite)

## 디자인/UX 규칙
- 기존 UI 스타일을 우선 복제해서 맞춘다(폰트/간격/배경 투명도/보더 스타일 등).
- 새로운 테마/과한 꾸밈(애니메이션, 페이지 추가, 모달 등)은 요청 없으면 추가하지 않는다.
- 좌측은 효과 탭, 우측은 컨트롤 패널 같은 **기존 구조를 유지**한다.

## 서버/포트 규칙
- 이미 사용 중인 포트가 있으면 자동으로 다른 포트를 사용.
- 가능하면 기존에 쓰던 포트(예: 5178)가 살아있으면 재사용하고, 충돌 시 +1 방식으로 회피.

## 파일/폴더 구조 규칙
- 새 효과는 `particles/<effect-id>/` 형태로 만든다.
- 새 효과는 기본적으로 `particles/image-metaballs/`를 복제해서 시작하고, 차이점은 최소 변경으로 유지.
- 경로는 상대경로 유지(정적 서버 기준).

## 완료 기준(각 작업 후)
- 페이지가 200 OK로 열리는지(최소 `curl -I` 수준) 확인.
- 콘솔 에러가 명백히 나올 만한 누락(ID mismatch 등)이 없도록 점검.
